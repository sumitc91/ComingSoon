65. Valid Number

Hard

Problem Description

A valid number can be split up into these components (in order):

  1. A decimal number or an integer.
  2. (Optional) An 'e' or 'E', followed by an integer.

A decimal number can be split up into these components (in order):

  1. (Optional) A sign character (either '+' or '-').
  2. One of the following formats:
    • One or more digits, followed by a dot '.'.
    • One or more digits, followed by a dot '.', followed by one or more digits.
    • A dot '.', followed by one or more digits.

An integer can be split up into these components (in order):

  1. (Optional) A sign character (either '+' or '-').
  2. One or more digits.

Given a string s, return true if s is a valid number.

Examples

Example 1:
Input: s = "0"
Output: true

Example 2:
Input: s = "e"
Output: false

Example 3:
Input: s = "."
Output: false

Example 4:
Input: s = "3."
Output: true

Example 5:
Input: s = "2e10"
Output: true

Python Solution


def isNumber(s: str) -> bool:
    # Define states and their transitions
    states = {
        0: {'digit': 1, 'sign': 2, 'dot': 3},
        1: {'digit': 1, 'dot': 4, 'e': 5},
        2: {'digit': 1, 'dot': 3},
        3: {'digit': 4},
        4: {'digit': 4, 'e': 5},
        5: {'digit': 7, 'sign': 6},
        6: {'digit': 7},
        7: {'digit': 7}
    }
    
    # Define final states
    final_states = {1, 4, 7}
    
    # Current state
    curr_state = 0
    
    for c in s.lower():
        if c.isdigit():
            c_type = 'digit'
        elif c in ['+', '-']:
            c_type = 'sign'
        elif c in ['e']:
            c_type = 'e'
        elif c == '.':
            c_type = 'dot'
        else:
            return False
        
        if c_type not in states[curr_state]:
            return False
        
        curr_state = states[curr_state][c_type]
    
    return curr_state in final_states

Java Solution


class Solution {
    public boolean isNumber(String s) {
        boolean seenDigit = false;
        boolean seenExponent = false;
        boolean seenDot = false;
        
        for (int i = 0; i < s.length(); i++) {
            char c = s.charAt(i);
            
            if (Character.isDigit(c)) {
                seenDigit = true;
            }
            else if (c == '+' || c == '-') {
                if (i > 0 && s.charAt(i-1) != 'e' && s.charAt(i-1) != 'E') {
                    return false;
                }
            }
            else if (c == 'e' || c == 'E') {
                if (seenExponent || !seenDigit) {
                    return false;
                }
                seenExponent = true;
                seenDigit = false;
            }
            else if (c == '.') {
                if (seenDot || seenExponent) {
                    return false;
                }
                seenDot = true;
            }
            else {
                return false;
            }
        }
        
        return seenDigit;
    }
}

Complexity Analysis

  • Time Complexity: O(n) where n is the length of the string
  • Space Complexity: O(1) as we only use a few boolean variables

Solution Explanation

This solution uses a state tracking approach to validate numbers:

  • Key components to track:
    • Digits (must have at least one)
    • Decimal point (at most one)
    • Exponent (at most one)
    • Signs (only at start or after exponent)
  • Validation rules:
    • Must see at least one digit
    • Decimal point can't appear after exponent
    • Exponent must have digits before it
    • Signs only valid at start or after exponent

Key points:

  • Handle all edge cases
  • Track state with boolean flags
  • Careful order of validation
  • Consider all valid number formats
