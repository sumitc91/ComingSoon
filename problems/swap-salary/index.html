627. Swap Salary

Jump to Solution: MySQL PostgreSQL Oracle MS SQL Server C#

Problem Description

Table: Salary 

+-------------+----------+
| Column Name | Type     |
+-------------+----------+
| id          | int      |
| name        | varchar  |
| sex         | ENUM     |
| salary      | int      |
+-------------+----------+
id is the primary key (column with unique values) for this table.
The sex column is ENUM value of type ('m', 'f').
The table contains information about an employee.

Write a solution to swap all 'f' and 'm' values (i.e., change all 'f' values to 'm' and vice versa) with a single update statement and no intermediate temporary tables.

Note that you must write a single update statement, do not write any select statement for this problem.

Example:

Input:

Salary table:
+----+------+-----+--------+
| id | name | sex | salary |
+----+------+-----+--------+
| 1  | A    | m   | 2500   |
| 2  | B    | f   | 1500   |
| 3  | C    | m   | 5500   |
| 4  | D    | f   | 500    |
+----+------+-----+--------+

Output:

+----+------+-----+--------+
| id | name | sex | salary |
+----+------+-----+--------+
| 1  | A    | f   | 2500   |
| 2  | B    | m   | 1500   |
| 3  | C    | f   | 5500   |
| 4  | D    | m   | 500    |
+----+------+-----+--------+

MySQL Solution

UPDATE Salary
SET sex = CASE sex
    WHEN 'm' THEN 'f'
    ELSE 'm'
END;

Alternative Solution using IF:

UPDATE Salary
SET sex = IF(sex = 'm', 'f', 'm');

C# Solution

public class Solution {
    public void Execute(string connectionString) {
        string sql = @"
            UPDATE Salary
            SET sex = CASE sex
                WHEN 'm' THEN 'f'
                ELSE 'm'
            END;";
            
        // Using Dapper
        using (var connection = new SqlConnection(connectionString)) {
            connection.Execute(sql);
        }
    }
}

Alternative Solution using Entity Framework:

public class Solution {
    public void Execute(SalaryContext context) {
        var employees = context.Salaries.ToList();
        foreach (var employee in employees) {
            employee.Sex = employee.Sex == "m" ? "f" : "m";
        }
        context.SaveChanges();
    }
    
    // Entity Framework model
    public class Salary {
        public int Id { get; set; }
        public string Name { get; set; }
        public string Sex { get; set; }
        public int SalaryAmount { get; set; }
    }
}

Implementation Notes:

  • Both raw SQL and Entity Framework approaches are shown
  • Raw SQL is more efficient for bulk updates
  • Entity Framework approach allows for more complex business logic
  • Both solutions maintain data integrity
