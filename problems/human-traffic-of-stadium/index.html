601. Human Traffic of Stadium

Problem Description

Table: Stadium 

+---------------+---------+
| Column Name   | Type    |
+---------------+---------+
| id           | int     |
| visit_date   | date    |
| people       | int     |
+---------------+---------+
id is the primary key for this table.
Each row of this table contains the visit date and visit id to the stadium with the number of people during the visit.
No two rows will have the same visit_date, and as the id increases, the dates increase as well.

Write an SQL query to display the records with three or more rows with consecutive id's, and the number of people is greater than or equal to 100 for each.

Return the result table ordered by visit_date in ascending order.

Example:

Input:

Stadium table:
+------+------------+-----------+
| id   | visit_date | people    |
+------+------------+-----------+
| 1    | 2017-01-01 | 10       |
| 2    | 2017-01-02 | 109      |
| 3    | 2017-01-03 | 150      |
| 4    | 2017-01-04 | 99       |
| 5    | 2017-01-05 | 145      |
| 6    | 2017-01-06 | 1455     |
| 7    | 2017-01-07 | 199      |
| 8    | 2017-01-09 | 188      |
+------+------------+-----------+

Output:

+------+------------+-----------+
| id   | visit_date | people    |
+------+------------+-----------+
| 5    | 2017-01-05 | 145      |
| 6    | 2017-01-06 | 1455     |
| 7    | 2017-01-07 | 199      |
| 8    | 2017-01-09 | 188      |
+------+------------+-----------+

Explanation:

  • The four rows with ids 5, 6, 7, and 8 have consecutive ids and each of them has >= 100 people attended.
  • Note that row 8 was included even though the visit_date was not the next day after row 7.
  • The rows with ids 2 and 3 are not included because we need at least three consecutive ids.

Solution

MySQL Solution

WITH ConsecutiveVisits AS (
    SELECT 
        id,
        visit_date,
        people,
        LAG(people, 1) OVER (ORDER BY id) prev1,
        LAG(people, 2) OVER (ORDER BY id) prev2,
        LEAD(people, 1) OVER (ORDER BY id) next1,
        LEAD(people, 2) OVER (ORDER BY id) next2
    FROM Stadium
)
SELECT DISTINCT s1.id, s1.visit_date, s1.people
FROM ConsecutiveVisits s1
WHERE 
    s1.people >= 100 AND (
        (s1.prev1 >= 100 AND s1.prev2 >= 100) OR
        (s1.prev1 >= 100 AND s1.next1 >= 100) OR
        (s1.next1 >= 100 AND s1.next2 >= 100)
    )
ORDER BY visit_date;

Approach Explanation

The solution uses window functions to find consecutive visits:

  1. Key Insights:
    • Window functions
    • LAG and LEAD
    • Consecutive IDs
    • Pattern matching
  2. Query Steps:
    • Get previous values
    • Get next values
    • Filter conditions
    • Order results

Implementation Details:

  • CTE usage
  • Window functions
  • Pattern matching
  • Result ordering

Optimization Insights:

  • Single pass
  • No self-joins
  • Efficient filtering
  • Index usage

Edge Cases:

  • Empty table
  • Gaps in IDs
  • Boundary cases
  • Date discontinuity
