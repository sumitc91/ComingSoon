Problem Description

There is only one character 'A' on the screen of a notepad. You can perform one of two operations on this notepad for each step:

  • Copy All: You can copy all the characters present on the screen (a partial copy is not allowed).
  • Paste: You can paste the characters which are copied last time.

Given an integer n, return the minimum number of operations to get the character 'A' exactly n times on the screen.

Examples:

Example 1:

Input: n = 3
Output: 3
Explanation: Initially, we have one character 'A'.
In step 1, we use Copy All operation.
In step 2, we use Paste operation to get 'AA'.
In step 3, we use Paste operation to get 'AAA'.

Example 2:

Input: n = 1
Output: 0
Explanation: We already have one 'A' on screen.

Constraints:

  • 1 ≤ n ≤ 1000

Python Solution

class Solution:
    def minSteps(self, n: int) -> int:
        # Base case: no operations needed for n = 1
        if n == 1:
            return 0
        
        # Find the smallest factor of n
        for i in range(2, int(n ** 0.5) + 1):
            if n % i == 0:
                # If we find a factor, we can:
                # 1. Get i A's using minSteps(i) operations
                # 2. Copy All once
                # 3. Paste (n/i - 1) times
                return i + self.minSteps(n // i)
        
        # If n is prime, we need n operations:
        # 1. Copy All once
        # 2. Paste (n-1) times
        return n

Alternative Solution (Dynamic Programming):

class Solution:
    def minSteps(self, n: int) -> int:
        dp = [0] * (n + 1)  # dp[i] represents min steps to get i A's
        
        for i in range(2, n + 1):
            dp[i] = i  # Worst case: i steps (Copy All once, then Paste i-1 times)
            
            # Try to find a factor j where we can:
            # 1. Get j A's
            # 2. Copy All once
            # 3. Paste (i/j - 1) times
            for j in range(2, int(i ** 0.5) + 1):
                if i % j == 0:
                    dp[i] = min(dp[i], dp[j] + (i // j))
                    dp[i] = min(dp[i], dp[i // j] + j)
        
        return dp[n]

Implementation Notes:

  • First solution uses recursion with prime factorization, O(sqrt(n)) time per recursive call
  • Second solution uses dynamic programming, O(n * sqrt(n)) time complexity
  • Both solutions exploit the fact that optimal solution involves factoring n

C++ Solution

class Solution {
public:
    int minSteps(int n) {
        vector dp(n + 1);
        
        for (int i = 2; i <= n; i++) {
            dp[i] = i;  // Worst case: i steps
            
            // Try to find factors
            for (int j = 2; j * j <= i; j++) {
                if (i % j == 0) {
                    // We can either:
                    // 1. Get j A's, then copy and paste i/j - 1 times
                    // 2. Get i/j A's, then copy and paste j - 1 times
                    dp[i] = min(dp[i], dp[j] + i/j);
                    dp[i] = min(dp[i], dp[i/j] + j);
                }
            }
        }
        
        return dp[n];
    }
};

var minSteps = function(n) {
    // Base case
    if (n === 1) return 0;
    
    // Find the smallest factor
    for (let i = 2; i <= Math.sqrt(n); i++) {
        if (n % i === 0) {
            return i + minSteps(n / i);
        }
    }
    
    // If n is prime
    return n;
};

Alternative Solution (Iterative):

/**
 * @param {number} n
 * @return {number}
 */
var minSteps = function(n) {
    let steps = 0;
    let d = 2;
    
    while (n > 1) {
        while (n % d === 0) {
            steps += d;
            n /= d;
        }
        d++;
    }
    
    return steps;
};

  • First solution uses recursion with prime factorization
  • Second solution uses iterative approach to find prime factors
  • Both solutions have O(sqrt(n)) time complexity

            for (int j = 2; j * j <= i; j++) {
                if (i % j == 0) {
                    dp[i] = Math.Min(dp[i], dp[j] + i/j);
                    dp[i] = Math.Min(dp[i], dp[i/j] + j);

    }
}

        int steps = 0;
