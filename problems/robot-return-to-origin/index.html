657. Robot Return to Origin

Jump to Solution: Python Java C++ JavaScript C#

Problem Description

There is a robot starting at the position (0, 0), the origin, on a 2D plane. Given a sequence of its moves, judge if this robot ends up at (0, 0) after it completes its moves.

You are given a string moves that represents the move sequence of the robot where moves[i] represents its ith move. Valid moves are:

  • 'R' (Right): The robot moves to the right by one unit
  • 'L' (Left): The robot moves to the left by one unit
  • 'U' (Up): The robot moves up by one unit
  • 'D' (Down): The robot moves down by one unit

Return true if the robot returns to the origin after it finishes all of its moves, or false otherwise.

Examples:

Example 1:

Input: moves = "UD"
Output: true
Explanation: The robot moves up once, and then down once. All moves have the same magnitude, so it ended up at the origin where it started. Therefore, we return true.

Example 2:

Input: moves = "LL"
Output: false
Explanation: The robot moves left twice. It ends up two units to the left of the origin. We return false because it is not at the origin at the end of its moves.

Constraints:

  • 1 ≤ moves.length ≤ 2 × 104
  • moves only contains the characters 'U', 'D', 'L' and 'R'

Python Solution

class Solution:
    def judgeCircle(self, moves: str) -> bool:
        # Count vertical and horizontal movements
        x = y = 0
        
        for move in moves:
            if move == 'U':
                y += 1
            elif move == 'D':
                y -= 1
            elif move == 'R':
                x += 1
            else:  # move == 'L'
                x -= 1
        
        # Return true if robot is back at origin
        return x == 0 and y == 0

Alternative Solution (Using Counter):

from collections import Counter

class Solution:
    def judgeCircle(self, moves: str) -> bool:
        count = Counter(moves)
        return count['U'] == count['D'] and count['L'] == count['R']

Implementation Notes:

  • First solution uses coordinate tracking with O(n) time and O(1) space
  • Second solution uses Counter with O(n) time and O(1) space
  • Both solutions are equally efficient for this problem

Java Solution

class Solution {
    public boolean judgeCircle(String moves) {
        int x = 0, y = 0;
        
        for (char move : moves.toCharArray()) {
            switch (move) {
                case 'U': y++; break;
                case 'D': y--; break;
                case 'R': x++; break;
                case 'L': x--; break;
            }
        }
        
        return x == 0 && y == 0;
    }
}

Alternative Solution (Using LINQ):

public class Solution {
    public bool JudgeCircle(string moves) {
        return moves.Count(c => c == 'U') == moves.Count(c => c == 'D') &&
               moves.Count(c => c == 'L') == moves.Count(c => c == 'R');
    }
}

Implementation Notes:

  • First solution uses coordinate tracking approach
  • Second solution uses LINQ for a more declarative approach
  • LINQ solution is more readable but less efficient
