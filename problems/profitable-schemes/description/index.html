879. Profitable Schemes

Jump to Solution: Python Java C++ JavaScript C#

Problem Description

There is a group of n members, and a list of various crimes they could commit. The ith crime generates a profit[i] and requires group[i] members to participate in it.

If a member participates in one crime, that member can't participate in another crime.

Let's call a profitable scheme any subset of these crimes that generates at least minProfit profit, and the total number of members participating in that subset of crimes is at most n.

Return the number of schemes that can be chosen. Since the answer may be very large, return it modulo 10⁹ + 7.

Examples:

Example 1:

Input: n = 5, minProfit = 3, group = [2,2], profit = [2,3]
Output: 2
Explanation: To make a profit of at least 3, the group could either commit crimes 0 and 1, or just crime 1.
In total, there are 2 schemes.

Example 2:

Input: n = 10, minProfit = 5, group = [2,3,5], profit = [6,7,8]
Output: 7
Explanation: To make a profit of at least 5, the group could commit any crimes, as long as they commit one.
There are 7 possible schemes: (0), (1), (2), (0,1), (0,2), (1,2), and (0,1,2).

Constraints:

  • 1 ≤ n ≤ 100
  • 0 ≤ minProfit ≤ 100
  • 1 ≤ group.length ≤ 100
  • 1 ≤ group[i] ≤ 100
  • profit.length == group.length
  • 0 ≤ profit[i] ≤ 100

Python Solution

class Solution:
    def profitableSchemes(self, n: int, minProfit: int, group: List[int], profit: List[int]) -> int:
        MOD = 10**9 + 7
        length = len(group)
        
        # dp[k][i][j] represents the number of schemes using first k crimes
        # with i members and at least j profit
        dp = [[[0] * (minProfit + 1) for _ in range(n + 1)] for _ in range(length + 1)]
        dp[0][0][0] = 1
        
        for k in range(1, length + 1):
            members_needed = group[k-1]
            profit_gained = profit[k-1]
            
            for i in range(n + 1):
                for j in range(minProfit + 1):
                    # Don't commit this crime
                    dp[k][i][j] = dp[k-1][i][j]
                    
                    # Commit this crime if we have enough members
                    if i >= members_needed:
                        dp[k][i][j] += dp[k-1][i-members_needed][max(0, j-profit_gained)]
                    
                    dp[k][i][j] %= MOD
        
        return sum(dp[length][i][minProfit] for i in range(n + 1)) % MOD

Implementation Notes:

  • Uses 3D dynamic programming
  • Time complexity: O(length * n * minProfit)
  • Space complexity: O(length * n * minProfit)

Java Solution

class Solution {
    public int profitableSchemes(int n, int minProfit, int[] group, int[] profit) {
        final int MOD = 1_000_000_007;
        int length = group.length;
        
        // dp[k][i][j] represents the number of schemes using first k crimes
        // with i members and at least j profit
        int[][][] dp = new int[length + 1][n + 1][minProfit + 1];
        dp[0][0][0] = 1;
        
        for (int k = 1; k <= length; k++) {
            int members = group[k-1];
            int earn = profit[k-1];
            
            for (int i = 0; i <= n; i++) {
                for (int j = 0; j <= minProfit; j++) {
                    // Don't commit this crime
                    dp[k][i][j] = dp[k-1][i][j];
                    
                    // Commit this crime if we have enough members
                    if (i >= members) {
                        dp[k][i][j] = (dp[k][i][j] + dp[k-1][i-members][Math.max(0, j-earn)]) % MOD;
                    }
                }
            }
        }
        
        int sum = 0;
        for (int i = 0; i <= n; i++) {
            sum = (sum + dp[length][i][minProfit]) % MOD;
        }
        return sum;
    }
}
