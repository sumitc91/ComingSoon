602. Friend Requests II: Who Has the Most Friends

Problem Description

Table: RequestAccepted 

+----------------+---------+
| Column Name    | Type    |
+----------------+---------+
| requester_id   | int     |
| accepter_id    | int     |
| accept_date    | date    |
+----------------+---------+
(requester_id, accepter_id) is the primary key for this table.
This table contains the ID of the user who sent the request, the ID of the user who received the request, and the date when the request was accepted.

Write an SQL query to find the people who have the most friends.

Return the result table with the following columns:

  • id: The ID of the user
  • num: The total number of friends the user has

Return the result table ordered by id in ascending order.

Example:

Input:

RequestAccepted table:
+--------------+-------------+-------------+
| requester_id | accepter_id | accept_date |
+--------------+-------------+-------------+
| 1            | 2           | 2016/06/03  |
| 1            | 3           | 2016/06/08  |
| 2            | 3           | 2016/06/08  |
| 3            | 4           | 2016/06/09  |
+--------------+-------------+-------------+

Output:

+----+-----+
| id | num |
+----+-----+
| 3  | 3   |
+----+-----+

Explanation:

  • User 1 has 2 friends (2 and 3)
  • User 2 has 2 friends (1 and 3)
  • User 3 has 3 friends (1, 2 and 4)
  • User 4 has 1 friend (3)
  • User 3 has the most friends, with 3 friends.

Solution

MySQL Solution

WITH AllFriends AS (
    SELECT requester_id AS id FROM RequestAccepted
    UNION ALL
    SELECT accepter_id AS id FROM RequestAccepted
)
SELECT 
    id,
    COUNT(*) AS num
FROM AllFriends
GROUP BY id
HAVING COUNT(*) = (
    SELECT COUNT(*) AS cnt
    FROM AllFriends
    GROUP BY id
    ORDER BY cnt DESC
    LIMIT 1
)
ORDER BY id;

MS SQL Server Solution

WITH AllFriends AS (
    SELECT requester_id AS id FROM RequestAccepted
    UNION ALL
    SELECT accepter_id AS id FROM RequestAccepted
)
SELECT 
    id,
    COUNT(*) AS num
FROM AllFriends
GROUP BY id
HAVING COUNT(*) = (
    SELECT TOP 1 COUNT(*) AS cnt
    FROM AllFriends
    GROUP BY id
    ORDER BY cnt DESC
)
ORDER BY id;

Approach Explanation

The solution uses a combination of UNION ALL and aggregation:

  1. Key Insights:
    • Bidirectional friendships
    • Count aggregation
    • Maximum finding
    • Result filtering
  2. Query Steps:
    • Combine friend lists
    • Count frequencies
    • Find maximum
    • Filter results

Implementation Details:

  • CTE usage
  • UNION ALL
  • GROUP BY
  • HAVING clause

Optimization Insights:

  • Single scan
  • No self-joins
  • Efficient grouping
  • Index usage

Edge Cases:

  • Empty table
  • Single friend
  • Multiple maximums
  • Duplicate requests
